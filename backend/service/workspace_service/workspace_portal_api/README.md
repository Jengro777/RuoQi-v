# 工作区管理门户

外部用户的主入口。登录后切换工作区，通过工作区使用各租户的应用。跨租户切换对用户透明——切换工作区即切换租户上下文。

## 用户旅程

```
外部用户注册 → 自动获取默认工作区 → 进入工作区 → 使用应用
```

## 核心约束

- 一个工作区只绑定一个租户
- 一个租户可创建多个工作区，每个工作区分配该租户已订阅应用的不同子集
- 工作区内可使用的应用全部来自该租户的订阅
- 任何外部用户均可创建工作区，创建时选定一个租户（前提是该租户已邀请该用户加入）

## 默认工作区（消费者侧）

面向消费者的终端（买家、司机、货主等），系统在用户注册时自动完成：

```
注册 → 自动加入宿主租户 → 自动创建默认工作区 → 分配对应应用
```

- Mall 买家注册 → 自动加入 Mall 宿主租户 → 默认工作区（只分配买家应用）
- TMS 司机注册 → 自动加入 TMS 宿主租户 → 默认工作区（只分配司机应用）
- WMS 货主注册 → 自动加入 WMS 宿主租户 → 默认工作区（只分配货主应用）

宿主租户由平台运营方维护，对用户无感知。用户之后若被企业租户邀请加入，可额外创建绑定企业租户的工作区，通过切换工作区在个人身份和企业身份间切换。

## 业务边界

- 外部用户主页面：查看自己加入的所有工作区（可跨多个租户），切换进入
- 工作区内：展示当前工作区所属租户订阅了哪些应用，点击跳转使用
- 创建/管理工作区
- 将租户已订阅的应用分配到本工作区
- 工作区成员与权限管理（成员从该租户的成员池中选择）
- 不执行具体业务操作（实际使用跳转到域内角色门户）

## 账户模型

- 外部用户由平台统一注册，可加入多个工作区（分别属于不同租户）
- 切换工作区 = 切换租户上下文，用户无需感知租户概念
- 中台账号不登录此处（见 `platform_service/platform_portal_api`）

## 注意

与 `platform_service/platform_workspace_portal_api` 区分：这里是工作区成员自己操作，中台管理员的管理在 `platform_service` 下。
