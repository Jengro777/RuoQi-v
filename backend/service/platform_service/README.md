# 中台服务

中台运营者管理全局资源的服务模块（HTTP + RPC）。

## 设计动机

统一外部用户账号体系。外部用户在平台上只有一个账号，跨 Mall、TMS、WMS 等产品统一登录，通过切换工作区进入不同租户上下文。避免多产品各自建账号体系造成的重复注册或 OAuth 联邦复杂度。

## 账户模型

- **中台账号**：仅用于登录中台管理面，只能在中台内部开通，与外部账号体系物理隔离
- **外部账号**：用户在各终端首次注册时创建，全平台唯一，通过加入工作区访问各租户应用

## 宿主租户

每个产品（Mall、TMS、WMS）下设有宿主租户，由平台运营方维护。消费者侧用户（买家、司机、货主等）注册时自动加入对应产品的宿主租户并获取默认工作区。宿主租户对终端用户透明，不暴露在用户界面。

## 租户数据隔离

同时支持两种隔离策略，按租户级别灵活选择：

### 库级隔离（database-per-tenant）

高安全等级租户分配独立数据库，租户间数据物理隔离。业务表无需 `tenant_id` 字段，回归单租户设计。

优势：数据不可能串库；备份、恢复、迁移均按租户粒度操作。

### 表级隔离（共享数据库 + tenant_id）

普通租户共享数据库，每张业务表携带 `tenant_id` 字段，查询时由中间件注入过滤条件。

### 路由机制

```
请求 → workspace_id → tenant_id → 查询租户元数据获取 DB 连接串 → 路由到对应库
```

隔离策略对业务代码透明。域内服务不感知底层是独立库还是共享库，只拿到一个 DB 连接。租户创建时，中台根据租户等级决定分配独立数据库还是加入共享库。

## 业务边界

- 管理所有租户（含宿主租户和企业租户）、工作区、用户、应用
- 不涉及具体业务域（商城、运输、仓储）的运营操作
